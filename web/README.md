# 电工模拟器 · 网页原型（单灯 / 点动 / 自锁）

端到端原型：在浏览器里选练习、拖元件、接线、运行，看灯亮/电机转、看判错。
界面用 Vite + React + React Flow，逻辑直接复用引擎（`../engine/src/`）。

## 运行

```bash
cd 电工模拟器-web
npm install
npm run dev      # 打开 http://localhost:5173/
npm test         # 运行 web 层集成测试（自锁持久化 / 点动）
```

> 若提示 esbuild 脚本被拦截，执行一次 `npm rebuild esbuild` 再 `npm run dev`。

## 三个练习（左侧切换）

1. **单灯单控**：接 L → 开关 → 灯泡 → N，点开关让灯亮灭。
2. **接触器点动**：控制回路 L1→停止(常闭)→启动(常开)→线圈→N；主回路 三相→主触点→电机。
   **按住**启动电机转，**松开即停**。
3. **接触器自锁**：在启动按钮上**并联**接触器辅助常开触点。按启动后松开仍保持运行，按停止才停。

操作：拖端子小圆点接线（端子不分方向）；开关点击切换；按钮按住生效、松开复位。
点「🔍 检查电路」对照练习模板做语义判错（如停止按钮类型、自锁触点是否并联）。

## 与引擎的关系

- 引擎不再复制：web 直接以相对路径引用 `../engine/src/`，单一来源，不会漂移。
- `App.tsx` 的 `buildCircuit()` 把 React Flow 的节点/连线翻译成引擎的 `Circuit`，
  调 `simulate()`，再把 `working / errors / shorts` 映射回界面。

## 这条竖切验证了什么

- ✅ 引擎能无改动接进网页（同一份 TypeScript）。
- ✅ 拖拽接线 + 点击运行 + 实时反馈的主链路跑通。
- 待你在浏览器里手感确认：接线的顺滑度、端子是否好点中。
